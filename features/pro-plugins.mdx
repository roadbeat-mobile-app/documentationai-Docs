title: Pro Plugins
description: How the roadbeat Mobile App integrates with Studio Pro plugins — remote API access, Web Component hosting, and license detection.

The roadbeat Mobile App supports **Pro plugins** when connected to a Studio instance with an active Pro license. Pro features are only available in **remote mode** — local mode operates with Community Edition features only.

## Pro Detection

On app startup (remote mode), the `ProDetectionService` checks the connected Studio for Pro availability:

```typescript
const manifest = await this.http.get<ProManifest>(
  `${studioUrl}/api/v1/pro/manifest`
);
```

The manifest contains:

| Field | Type | Description |
|-------|------|-------------|
| `licensed` | `boolean` | Whether the Studio has an active Pro license |
| `plugins` | `ProPlugin[]` | List of installed Pro plugins |
| `expiresAt` | `string?` | License expiration date |

## Available Pro Plugins

| Plugin | Description | Mobile Support |
|--------|-------------|----------------|
| **pro-workflows** | Editorial review workflows with approval steps | Via API — task list, approve/reject actions |
| **pro-scheduling** | Schedule content for future publication | Via API — date/time picker, schedule list |
| **pro-versioning** | Side-by-side version comparison with field diffs | Via API — version list, diff view |
| **pro-locking** | Content locking to prevent edit conflicts | Via API — lock indicator, lock/unlock |
| **pro-audit** | Activity audit trail for compliance | Via API — audit log viewer |
| **pro-import** | Bulk content import from CSV/JSON | Via API — import wizard |
| **pro-export** | Export content to various formats | Via API — export dialog |
| **pro-media** | Advanced media processing and editing | Via API + enhanced camera (mobile-pro) |
| **pro-realtime** | Real-time collaboration via WebSocket | WebSocket connection from mobile |
| **pro-notifications** | Advanced notification rules and routing | Via API — notification preferences |
| **pro-advanced-roles** | Fine-grained role-based access control | Via API — permission checks |
| **pro-sso** | Single Sign-On integration | Via API — SSO redirect flow |
| **pro-preview** | Live content preview in consumer context | Via API — preview renderer |
| **pro-whitelabel** | Custom branding for Studio | Theme applied to mobile app |
| **pro-saved-views** | Saved filter/sort configurations | Via API — saved view list, apply |
| **pro-api-explorer** | Interactive API documentation | Via API — endpoint browser |

## Plugin Host

The `ProPluginPage` component acts as a Web Component host for Pro plugin UIs:

```mermaid
graph TD
    ROUTE[/tabs/pro/:pluginId] --> HOST[PluginHostComponent]
    HOST --> DETECT{Plugin Available?}
    DETECT -->|Yes| LOAD[Load Web Component<br/>from Studio API]
    DETECT -->|No| ERROR[Show Unavailable Message]
    LOAD --> RENDER[Render Custom Element<br/>in Ionic Page]
```

Pro plugins ship their frontend as **Web Components** (Angular Elements) that are loaded dynamically from the Studio server. The mobile app:

1. Checks if the plugin is enabled in the manifest
2. Loads the plugin's JavaScript bundle from `${studioUrl}/api/v1/pro/plugins/${pluginId}/bundle.js`
3. Registers the custom element
4. Renders it inside an Ionic page with the app's theme

## Plugin API Access

For plugins that don't have a dedicated UI, the mobile app calls their REST API endpoints directly:

```typescript
// Example: Pro Scheduling — schedule content for future publish
await this.http.post(
  `${studioUrl}/api/v1/pro/scheduling/schedule`,
  { contentId, publishAt: '2026-04-15T10:00:00Z' }
);

// Example: Pro Workflows — approve a review task
await this.http.post(
  `${studioUrl}/api/v1/pro/workflows/tasks/${taskId}/approve`,
  { comment: 'Looks good!' }
);
```

## License Tiers

Pro features are gated by license tier:

| Tier | Plugins Included |
|------|-----------------|
| **Starter** | saved-views, export, import, preview |
| **Team** | + workflows, scheduling, locking, versioning, audit |
| **Business** | + advanced-roles, sso, whitelabel, realtime |
| **Enterprise** | + api-explorer, graphql, environments, custom plugins |

The mobile app checks `proLicenseTier` on the active account to show/hide Pro features in the UI.

## Local Mode Limitations

<Callout kind="alert">
  Pro plugins are **not available** in local mode. They require a running Studio server with a Pro license. In local mode, the Pro section of the app shows a message explaining this and offering to connect to a Studio instance.
</Callout>

Exceptions where Pro-like functionality works locally:

| Feature | Local Equivalent |
|---------|-----------------|
| **Versioning** | Local SQLite version snapshots |
| **Export** | Local file export |
| **Preview** | Local preview rendering |
| **Scheduling** | Simplified local scheduler |
| **Saved Views** | Local saved views in SQLite |
